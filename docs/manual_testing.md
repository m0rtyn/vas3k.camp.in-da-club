## Чеклист для тестирования

Исходя из реализованного (Phase 1 MVP), вот что нужно проверить:

### 1. Авторизация (dev-режим)
- [ ] Открыть http://localhost:5173 → редирект на `/login`
- [ ] Ввести username в dev-форме (например `alice`) → нажать «Войти» → попадаешь на Dashboard
- [ ] Открыть DevTools → Application → Local Storage → проверить, что сохранился токен
- [ ] Перезагрузить страницу → остаёшься залогиненным (fetchMe работает)
- [ ] Нажать «Выйти» → возврат на `/login`

### 2. Создание встречи (NFC-сценарий)
- [ ] Залогиниться как `alice`
- [ ] Перейти на `/:username` другого пользователя (например `/bob`) — но `bob` ещё не существует, поэтому сначала: открыть второй браузер/инкогнито → залогиниться как `bob`
- [ ] Вернуться в первый браузер (`alice`) → перейти на `/bob` → увидеть профиль
- [ ] Нажать «Я встретил этого человека» → встреча создаётся
- [ ] Повторно зайти на `/bob` → должно показать «Уже встречались ✓»
- [ ] Попробовать создать встречу с самим собой (`/alice`) — должен быть другой UI (это твой профиль)

### 3. Отмена встречи (5-минутное окно)
- [ ] Создать встречу → сразу отменить → встреча исчезает, пара может встретиться снова
- [ ] Создать встречу → подождать >5 минут → кнопка отмены должна быть недоступна

### 4. Скрытие встречи
- [ ] Создать встречу → после 5 минут → нажать «Скрыть» → встреча пропадает из контактов
- [ ] Нажать «Показать скрытые» (если есть фильтр) → встреча видна
- [ ] «Показать обратно» → встреча снова в списке

### 5. Контакты (`/contacts`)
- [ ] Создать несколько встреч (залогиниться 3-4 разными пользователями в инкогнито) 
- [ ] Открыть `/contacts` → увидеть список всех встреч
- [ ] Проверить фильтры: все / confirmed / unconfirmed
- [ ] Встречи должны быть видны **обоим** участникам

### 6. Лидерборд (`/leaderboard`)
- [ ] Открыть `/leaderboard` → увидеть ранжирование по confirmed meetings
- [ ] Свою позицию видно, остальные анонимизированы (за пределами топ-20)

### 7. Dashboard (`/`)
- [ ] Показывает количество встреч, последние контакты, счётчик approvals

### 8. Админка (`/admin`)
- [ ] Залогиниться пользователем с `is_admin = true` (нужно вручную поставить в БД: `UPDATE users SET is_admin = true WHERE username = 'alice'`)
- [ ] Открыть `/admin` → выдать approvals конкретному пользователю
- [ ] Выдать approvals всем (`__all__`)
- [ ] Посмотреть статистику

### 9. Оффлайн
- [ ] Открыть DevTools → Network → Offline
- [ ] Создать встречу → она сохраняется в IndexedDB
- [ ] Проверить, что показывается баннер «Нет сети, N действий в очереди»
- [ ] Включить сеть обратно → действия синхронизируются

### 10. Базовые edge-cases
- [ ] Двойной тап на кнопку «Meet» — не должно создать две встречи
- [ ] Одна пара — только одна активная встреча (уникальность A↔B)
- [ ] API отдаёт 401 без токена на защищённые роуты
